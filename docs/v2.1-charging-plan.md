# Glovebox v2.1 — Charging / EV-efficiency tracking

**Branch:** `dev/v2.1-charging`
**Status:** Draft for build
**Last updated:** 2026-07-20

Adds a **Charging** log for the Honda Clarity PHEV and reworks the efficiency card
so the headline number is **real gas MPG** (electric miles excluded) instead of the
inflated blended figure. Goal chosen with the user: *accurate efficiency*.

## Why this exists (the PHEV problem)

The v2 dashboard computes `miles ÷ gallons`. On a plug-in hybrid, a chunk of miles
are driven on the battery and burn **zero gas**, so that number is inflated (reads
~60+ when the engine's real efficiency is ~42 mpg). To correct it we need to know
how many miles were electric — which the car itself will not tell us (see below), so
we estimate them from the weekly charging session.

## Research finding: the car has no EV-miles counter

The 2019 Clarity PHEV has **no built-in lifetime/per-tank "EV miles driven" readout**
(owner consensus: [clarityforum](https://www.clarityforum.com/threads/how-to-find-my-total-electric-miles.2338/)).
It shows EV *range remaining*, trip meters (distance + **mpg**, not split by mode),
and a real-time energy screen — none of which is cumulative electric miles. So the
"read exact EV miles off the dash" upgrade floated earlier **is not available**, and
**ChargePoint's "miles added" is our best electric-miles proxy.** (Optional one-time
calibration via the trip-meter trick — see §6.)

## 1. What we track

**Existing — `fuel_entries`** (unchanged): `date`, `odometer` (mi), `volume` (gal).

**New — `charging_sessions`** (`migrations/0004_charging.sql`):

```sql
CREATE TABLE charging_sessions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  date TEXT NOT NULL,                    -- YYYY-MM-DD
  odometer INTEGER NOT NULL,             -- MILES (canonical); aligns a charge to a fuel window
  kwh REAL NOT NULL,                     -- energy delivered (ChargePoint) — exact
  miles_added REAL,                      -- ChargePoint's estimated range added — electric-miles proxy
  notes TEXT,
  added_by TEXT NOT NULL,                -- from Cf-Access-Authenticated-User-Email
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_charging_date ON charging_sessions (date DESC, id DESC);
```

No cost/price fields (charging is free; the "$ saved" metric was dropped).

## 2. Logging flow (UX)

The fuel and charging logs share **one tab — "Fuel & Charging"** (route `#/log`, the
default landing) — behind a **Gas / Charge segmented control**. The Gas segment shows
the existing fuel form + fuel history; the Charge segment shows an inline charge form
+ charging history. The blended efficiency card is pinned **above** the control
(always visible). Charging is **add-only with delete** (no edit, like Fuel); there is
no separate "Add charge" button — the form is inline, like the fill-up form.

Both panels stay in the DOM and are shown/hidden, so a half-typed entry survives a
segment switch; the last-used segment is remembered in `localStorage`. After the
weekly ChargePoint session, flip to **Charge** and enter **odometer** (dash), **kWh**
and **miles** (ChargePoint), notes optional; date defaults to today. ~15 seconds/week.
`odometer` and `miles_added` follow the mi/km toggle at the UI edge; stored canonical.

## 3. API

| Method | Path | Purpose |
|---|---|---|
| GET | `/glovebox/api/charging` | list, `ORDER BY date DESC, id DESC` |
| POST | `/glovebox/api/charging` | create (`added_by` server-derived) |
| DELETE | `/glovebox/api/charging/:id` | delete |

Validation: `date` + `odometer` + `kwh` required and numeric (`kwh > 0`);
`miles_added` optional numeric. Same Access-header → `LOCAL_DEV_EMAIL` → 401 pattern.

## 4. Metrics & exact calculations

Computed over the **fuel-fill-to-fuel-fill window** (needed for gas MPG), folding in
charging by odometer. Sort fuel entries by odometer; need ≥2.

```
span_miles      = odometer(last fuel)  − odometer(first fuel)
gallons_used    = Σ gallons of every fuel-up AFTER the first      // each fill tops the tank
electric_miles  = Σ miles_added of charges whose odometer ∈ [first, last fuel odometer]
kwh_used        = Σ kwh of those same charges

electric_miles  = min(electric_miles, span_miles)                 // guard against bad data
gas_miles       = span_miles − electric_miles
```

**Displayed:**
```
Real gas MPG  = gas_miles / gallons_used        ← HEADLINE (should land ≈ 42)
% electric    = electric_miles / span_miles × 100
mi/kWh (est.) = electric_miles / kwh_used         ← secondary, labeled "est." (see §6)
```

`mi/kWh` here re-derives ChargePoint's own conversion factor (~2.8–3), so it's an
estimate, not an independent measurement — hence the "est." tag.

**Worked example:** span 620 mi, gallons 11.0, in-window charges → miles_added 118,
kWh 42 → gas_miles 502 → **gas MPG ≈ 45.6**, **% electric ≈ 19%**, mi/kWh ≈ 2.8.

**Edge cases:** <2 fuel-ups → "log more fill-ups" (unchanged). No charges in window →
electric_miles 0 → gas MPG = the raw blended number (graceful). Bad data
(electric > span) → clamped, never a negative MPG.

## 5. Display

The Fuel-tab efficiency card (default landing) becomes:
- **Big:** Real gas MPG (label switches to **"Gas Efficiency"** once charging data
  exists; stays **"Average Efficiency"** before any charge is logged, since without
  charging the number is still blended/uncorrected).
- **Sub:** `19% electric · 2.8 mi/kWh (est) · 620 mi · 3 charges`.

Units toggle converts gas MPG (→ km/L) and mi/kWh (→ km/kWh); % electric is unitless.

## 6. Accuracy & optional calibration

- gas MPG and % electric depend on ChargePoint's range figure standing in for
  electric miles → good **estimates for trends**, not lab-exact. MPGe (dropped from
  the display) is the only split-free exact metric, computable later if wanted.
- AC charging loses ~10–15% as heat, so kWh delivered > energy to the wheels; mi/kWh
  reads a touch low. Consistent, not wrong.
- Best read over long windows; week-to-week noise (cold starts, hard accel burning
  some gas on a charge) averages out.
- **Optional one-time calibration:** do the trip-meter trick once — zero Trip B on a
  full charge, note the miles when the battery depletes = real EV miles for that
  charge; divide by that session's kWh = *your* true mi/kWh. Use it to sanity-check
  or replace ChargePoint's factor. No recurring effort.
